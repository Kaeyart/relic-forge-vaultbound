# Patch 004 — UI Rebuild

## Goal

Replace the messy prototype overlay feeling with a cleaner combat-first UI.

## Design rules

- The center of the screen belongs to gameplay.
- Core combat info stays compact and on the edges.
- Detailed information lives in a right-side drawer.
- Choice moments get a centered, readable selection sheet.
- Old debug labels should stay hidden.

## What this patch adds

- Minimalist bottom-left status block
- Minimalist bottom-center action bar
- Top-center room chip
- Bottom-right interaction prompt
- Right-side drawer for Build / Inventory / Skill Tree / Dungeon
- Centered choice sheet for rewards, route selection, passives, and skill drafts

## What this patch does not solve yet

- Fancy icons
- Final typography
- Final item comparison cards
- Final passive tree layout

This patch is about structure and readability first.
