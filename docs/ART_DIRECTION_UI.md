# Relic Forge: Vaultbound — UI / Asset Direction V1

## Core goal

The UI and static asset language should make the game feel like a real buildcraft ARPG, not a debug prototype.

## Style keywords

- dark forge fantasy
- relic-tech
- occult machinery
- practical, readable, restrained
- not cartoon
- not glossy mobile
- not neon sci-fi overload
- not overdecorated gothic clutter

## Material language

- blackened iron
- charcoal panels
- worn brass trims
- ember-orange highlights
- void-purple accents
- mana / cold / lightning blue-white accents
- subtle texture, not noisy texture
- thin borders over thick frames

## UI philosophy

- low intrusion
- combat readability first
- strong spacing
- clear hierarchy
- compact bars and chips
- avoid giant full-screen overlays during active play
- world interaction should stay physical when possible

## Color use

### Primary neutrals
- dark charcoal
- iron black
- smoke grey

### Accent families
- Health / fire / danger: ember orange to hot red
- Mana / cold / storm: pale blue to electric cyan
- Void / cursed / arcane: violet to purple
- Rare / reward / gold: brass / muted gold
- Selection / active: warm pale gold or pale arcane glow

## First asset groups

1. Panels / prompts / slots / borders
2. Skill icons
3. Damage tag icons
4. Equipment slot icons
5. Material icons
6. Hub activity icons
7. Static VFX sprites
