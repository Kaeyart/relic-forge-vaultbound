# Design Lock — Combat and Build Identity

Skills should have mechanical identity first. They should not be colored copies of one another.

## Skill identity

Each active skill needs a clear role:

- projectile;
- area control;
- melee burst;
- trap setup;
- curse/rift setup;
- chaining hit.

Elemental conversion should change gameplay when possible, not just tint the numbers.

## Synergy and procs

Builds should be built from interactions:

- Skill A applies a state.
- Skill B consumes that state.
- An item or support gem changes the rule.
- A passive or unique turns that into a build engine.

## Readability

Deep synergy is allowed, but combat must remain readable. Effects need distinct shapes, timing, and colors so the player can understand what is happening.
